单一职责原则

开放封闭原则

比如“一国两制”政策，社会主义制度不能修改，但这样回归会困难重重，所以增加了一种制度，“一国两制”是政治上的伟大发明；在软件设计模式中，这种不能修改，但可以扩展的思想也是一种重要的设计原则，即开闭原则。

概念

对于扩展是开放的，对于更改是封闭的。软件实体（类、模块、函数等）可以扩展，但不能修改。

  • 这样在面对新需求时，不必把整个程序推倒重来，对程序的改动是通过增加新代码进行的，而不是更改现有代码。

依赖倒转原则

依赖倒转可以说是面向对象设计的标志。依赖倒转其实就是谁也不要依靠谁，除了约定的接口，大家都可以灵活自如。

概念

高层模块不应该依赖低层模块，两个模块都应该依赖抽象。

抽象不应该依赖细节，细节应该依赖于抽象。

  • 即针对接口编程，不要对实现编程。

里氏代换原则

概念

子类必须能够替换其父类。

  • 正是由于子类型的可替换性，才使得使用父类型的模块在无需修改的情况下就可以扩展。
